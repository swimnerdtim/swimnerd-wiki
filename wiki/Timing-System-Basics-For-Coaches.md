# Timing System Basics For Coaches

**Source:** CTS_Training_Manual (Colorado Timing System Operator Training)

## Why Coaches Should Understand Timing Systems

As a coach, understanding how electronic timing works helps you:
- **Troubleshoot meet delays** - Know when issues are fixable vs. systemic
- **Advocate for accurate times** - Understand what went wrong when results look suspect
- **Train relay exchanges** - Know reaction time standards and DQ thresholds
- **Plan meet-day logistics** - Work with timing operators to keep meets running smoothly

---

## Three Types of Timing Systems

**Per USA Swimming Rules (102.24.2):**

### 1. Automatic Timing
- **Started by:** Starting device
- **Stopped by:** Swimmer touching touchpad
- **Required for:** World, American, and US Open records
- **Accuracy:** Highest (pad activation = official time)

### 2. Semi-Automatic Timing
- **Started by:** Starting device  
- **Stopped by:** Buttons pushed by lane timers
- **Use case:** Most age-group and high school meets
- **Backup:** Still requires manual watches

### 3. Manual Timing
- **Started by:** Lane timer watching/hearing start
- **Stopped by:** Lane timer pressing stopwatch
- **Use case:** Dual meets, backup timing
- **Accuracy:** Lowest (human reaction time variance)

## Equipment Components

**Standard Meet Setup Includes:**

1. **CTS Console (Colorado Timing System)** - Central control unit
2. **Cable Harness** - Connects all components (pads, buttons, starting system)
3. **Starting System** - Microphone/horn sends electronic start signal
4. **Touchpads** - Mounted on wall at each lane (automatic timing)
5. **Buttons** - Hand-held devices pressed by lane timers (2-3 per lane)
6. **Printer** - Prints race results
7. **Scoreboard** - Displays times/splits

### Touchpad Connection Priority

- **"Prime"** connection = Touchpad
- **"Button A"** = First backup button
- **"Button B"** = Second backup button  
- **"Button C"** = Third backup button

**If only buttons (no pads):**
- Button 1 → "Prime" connection
- Button 2 → "Button A" connection

## How Times Are Recorded

### With Touchpads (Automatic)

**Primary time:** Touchpad activation  
**Backup times:** Buttons A, B, C

**Display on CTS Console:**
- **Two asterisks (\*\*)** = Both backup buttons recorded
- **One asterisk (\*)** = Only one button recorded
- **No asterisks** = No button backups (pad-only time)

### Button-Only (Semi-Automatic)

**Official time:** Average of buttons or middle time of three  
**Required:** Minimum 2 buttons per lane

**USA Swimming Rules:**
- If 2 buttons match within 0.3s → Use matching time
- If 3 buttons → Use middle time
- If buttons differ >0.3s → Manual watch becomes backup

## Common Timing Issues (Coach Perspective)

### 1. "Missed Touch" on Turns

**What happened:** Swimmer turned, but touchpad didn't register

**CTS Response:** Operator presses "+ TOUCH" to manually credit the length

**Coach tip:** Watch for "ARMED" indicator to reset after each turn. If it stays lit, the pad may have failed.

### 2. "Finish Armed" Doesn't Flash

**What happened:** System doesn't recognize final lap approaching

**CTS Response:** Operator presses "FINISH ARM" button for that lane

**Coach tip:** If a swimmer finishes but no time shows, this is likely the issue. Notify timing operator immediately.

### 3. Extra Touch Registered

**What happened:** Relay swimmer bumps pad after finishing, or swimmer double-touches

**CTS Response:** Operator presses "— TOUCH" to remove errant touch

**Coach tip:** In relays, have finished swimmers exit quickly to avoid pad contact.

### 4. Missed Start Signal

**What happened:** CTS console doesn't receive electronic start

**Display shows:** "RESET" instead of running clock

**CTS Response:** Operator manually presses "START" button

**Impact:** Times may be slightly off if start delay occurred

**Coach action:** If you suspect a missed start, alert referee immediately (before race finishes if possible).

## Relay-Specific Timing Considerations

### Relay Exchange Reaction Times

**How it's measured:**
- Outgoing swimmer's touchpad activation starts exchange timer
- Incoming swimmer's takeoff (detected by starting platform) stops timer
- **Legal range:** 0.00 to +0.30s
- **DQ threshold:** Negative reaction time (early takeoff)

**Displayed on CTS:**
- Each swimmer's exchange time shown with "r:" prefix
- Example: "r:+0.18" = 0.18s reaction time (legal)
- Example: "r:-0.05" = Early takeoff = **DISQUALIFICATION**

### Training Implication

**Optimal reaction time:** 0.10-0.25s
- Too slow (>0.30s) = Losing time unnecessarily
- Too fast (<0.00s) = DQ risk
- **Sweet spot:** 0.15-0.20s

**Practice drill:** Use timing system or video analysis to train consistent 0.15-0.20s reactions.

## What to Do When Times Look Wrong

### Step 1: Check the CTS Printout

Ask the timing operator for the heat sheet printout. It shows:
- Touchpad time (if applicable)
- Button times (A, B, C)
- Splits (if recorded)
- Reaction times (relays)

### Step 2: Compare Times

**Red flags:**
- Swimmer's time is 2+ seconds off their seed time (without obvious reason)
- Button times differ by >0.5s (suggests one timer missed finish)
- No touchpad time but swimmer clearly touched

### Step 3: File an Official Inquiry

If you believe the time is incorrect:
1. Notify the administrative official **immediately** (before next heat if possible)
2. Provide specific details (e.g., "Lane 4, button times show 58.7 and 59.8 but pad shows 1:03.2")
3. Official may review video, consult with referee, or request manual watch times

**Time limit:** Most meets require inquiries within 30 minutes of the race.

## Coach's Meet-Day Checklist

**Before Warm-Ups:**
- [ ] Confirm timing system is fully operational (ask timing operator)
- [ ] Verify touchpads are working in your assigned lanes
- [ ] Check scoreboard displays correctly

**During Competition:**
- [ ] Watch for "ARMED" indicators on turns (if visible)
- [ ] Note any lane timer issues (missed buttons)
- [ ] Record unofficial times on your own stopwatch for key races

**After Each Heat:**
- [ ] Compare CTS times to your watch times (should be within 0.3s)
- [ ] Flag any suspicious results immediately
- [ ] Save official heat sheets for post-meet review

## Terminology Quick Reference

| Term | Definition |
|------|------------|
| **ARMED** | Touchpad ready to record next touch |
| **FINISH ARMED** | System ready to record race finish |
| **Prime** | First/primary timing input (touchpad or button 1) |
| **Button A/B/C** | Backup button positions |
| **+ TOUCH** | Manually add a lap/length |
| **— TOUCH** | Remove an errant touch |
| **STORE/PRINT** | Save race results and print heat sheet |
| **RESET** | Clear system for next heat |

**Coach's Bottom Line:**

> You don't need to run the timing system, but understanding it makes you a better advocate for your swimmers. Know when to question results, how to work with officials, and what technical issues look like. A good coach knows the difference between a slow swim and a timing malfunction.

**Key Coaching Takeaway:**
- Touchpads are primary, buttons are backup
- Reaction times 0.15-0.20s are optimal for relays
- Question times immediately if they look wrong - don't wait until after the meet
